STEP 3 — PERSIST LICENSES (required — UI updates = redeploy = data wipe without this)
------------------------------------------------------------------------------------
WHY licenses disappear: Railway destroys the container disk on EVERY deploy.
Pushing UI changes triggers a new deploy → new empty disk → empty licenses.db.
This is NOT a bug in the panel code.

FIX (one-time):
1. Railway → license-server service → Volumes → Add Volume
2. Mount path: /data  (must be exactly /data)
3. Redeploy once (Railway sets RAILWAY_VOLUME_MOUNT_PATH=/data)
4. Deploy logs MUST show:
     [license-db] /data/licenses.db (... persistent=true)
     [license-db] Persistent volume: /data

Optional variable (only if mount path is not /data):
  DATABASE_PATH=/data/licenses.db

5. Re-register licenses ONCE (or use panel → Descargar respaldo / Restaurar if you have a JSON backup)

AdminPOS (site license — one per store, all cajas):
  - In POS server/.env set LICENSE_SITE_ID + LICENSE_KEY
  - In this panel, "Site ID" = same as LICENSE_SITE_ID (e.g. SITE-MI-TIENDA-001)
  - Do NOT register each cashier PC browser Machine ID for a 2-caja shop

BEFORE each deploy until volume exists: VenTECH panel → Descargar respaldo (JSON).
